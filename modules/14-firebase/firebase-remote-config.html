# üöÄ Remote Config

# Key Concepts

  • Remote Config Parameters: Key-value pairs stored in Firebase.
  • Defaults: Local default values to prevent fallback issues.
  • Fetching: Retrieving parameters from the Firebase backend.

# 1. Configuring Firebase Remote Config

# Set Default Values

Defaults provide a fallback for parameters that aren‚Äôt fetched from the backend.

import FirebaseRemoteConfig

func setupRemoteConfigDefaults() {
    let remoteConfig = RemoteConfig.remoteConfig()
    let defaults: [String: NSObject] = [
        "welcome_message": "Welcome to the app!" as NSObject,
        "max_items_to_display": 10 as NSObject
    ]
    remoteConfig.setDefaults(defaults)
}

# 2. Fetching and Activating Remote Config

The fetchAndActivate method allows you to fetch new values and immediately activate them.

# Using Async/Await

Here‚Äôs how to fetch remote parameters:

func fetchRemoteConfig() async throws {
    let remoteConfig = RemoteConfig.remoteConfig()
    
    do {
        // Setting a low fetch interval for testing
        let settings = RemoteConfigSettings()
        settings.minimumFetchInterval = 0
        remoteConfig.configSettings = settings
        
        // Asynchronous fetch and activate
        let status = try await remoteConfig.fetchAndActivate()
        
        switch status {
        case .successFetchedFromRemote:
            print("Config fetched from remote and activated.")
        case .successUsingPreFetchedData:
            print("Using previously fetched config.")
        case .error:
            print("Error activating fetched config.")
        @unknown default:
            print("Unknown activation status.")
        }
        
    } catch {
        print("Failed to fetch remote config: \(error.localizedDescription)")
    }
}

# 3. Retrieving Parameter Values

# Accessing Remote Config Parameters

Once activated, access parameters using RemoteConfigValue.

func getRemoteConfigValue() {
    let remoteConfig = RemoteConfig.remoteConfig()
    let welcomeMessage = remoteConfig["welcome_message"].stringValue ?? "Default Message"
    let maxItems = remoteConfig["max_items_to_display"].numberValue?.intValue ?? 0
    
    print("Welcome Message: \(welcomeMessage)")
    print("Max Items: \(maxItems)")
}

# 4. Using Remote Config with Firebase Authentication

Remote Config can tailor its values to individual users by leveraging Firebase Authentication.

# Customizing Parameters

To personalize parameters, use Remote Config Conditions set in the Firebase Console. Conditions can be based on user properties or audiences.

# Fetching Config After Login

Ensure Remote Config is fetched after authentication:

import FirebaseAuth

func authenticateAndFetchRemoteConfig() async throws {
    guard let user = Auth.auth().currentUser else {
        throw NSError(domain: "Authentication", code: 401, userInfo: [NSLocalizedDescriptionKey: "User not logged in"])
    }
    
    print("User ID: \(user.uid)")
    
    // Fetch Remote Config after authentication
    try await fetchRemoteConfig()
    getRemoteConfigValue()
}

# 7. Testing Tips

  • Use the Firebase Console to define default values.
  • Simulate different Remote Config conditions using the Remote Config Debug View.
  • Test with low fetch intervals during development.

# Enable Debug Mode

Debug mode ensures Remote Config fetches are not throttled:

settings.minimumFetchInterval = 0
