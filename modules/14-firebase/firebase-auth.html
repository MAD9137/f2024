# 🚀 Authentication

We’ll delve deeper into Firebase, focusing extensively on Firebase Authentication. We’ll enhance our approach by creating a dedicated service to manage all authentication operations and structure our app using the MVVM (Model-View-ViewModel) architecture. Additionally, we'll explore both Email/Password and Google Sign-In methods for user authentication.

# 1. Introduction to Firebase

Firebase is a comprehensive app development platform by Google that offers a suite of tools and services to help you build high-quality apps. Key Firebase services include:

  • Authentication: Secure user authentication.
  • Cloud Firestore: Flexible, scalable NoSQL cloud database.
  • Cloud Functions: Serverless backend.
  • Cloud Storage: Scalable file storage.
  • Crashlytics: Real-time crash reporting.
  • Analytics: Insights into user behavior.

# Why Firebase?

  1. Real-time Capabilities: Ideal for applications requiring real-time data synchronization.
  2. Scalability: Automatically scales with your app’s growth.
  3. Comprehensive SDKs: Provides robust SDKs with easy-to-use APIs for various platforms, including iOS.
  4. Security: Enhanced security features and easy integration with security rules.

# 2. Setting Up Firebase in Your SwiftUI Project

# Step 1: Create a Firebase Project

  1. Navigate to the Firebase Console (opens new window).
  2. Click on "Add project" and follow the prompts to create a new project.

# Step 2: Register Your iOS App

  1. In your Firebase project overview, click on the iOS icon to add an iOS app.
  2. Enter your app’s bundle ID (e.g., com.example.MyApp).
  3. Download the GoogleService-Info.plist file provided.
  4. Add the GoogleService-Info.plist file to your Xcode project by dragging it into the project navigator.

# Step 3: Install Firebase SDK via Swift Package Manager

  1. Open your Xcode project.
  2. Navigate to File > Add Packages...
  3. Enter the repository URL: https://github.com/firebase/firebase-ios-sdk
  4. Select the Firebase products you need (e.g., FirebaseAuth, FirebaseCore, FirebaseFirestore, FirebaseGoogleSignIn).
  5. Add the packages to your project.

# Step 4: Initialize Firebase

Update your App struct to configure Firebase at app launch.

import SwiftUI

class AppDelegate: NSObject, UIApplicationDelegate {
  func application(_ application: UIApplication,
                   didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey : Any]? = nil) -> Bool {

  }

# 3. Firebase Authentication Overview

Firebase Authentication provides backend services, easy-to-use SDKs, and ready-made UI libraries to authenticate users to your app. It supports authentication using passwords, phone numbers, and popular federated identity providers like Google, Facebook, and Apple.

# Authentication Methods:

  • Email and Password Authentication
  • Social Authentication (Google, Facebook, Apple, etc.)
  • Anonymous Authentication
  • Phone Authentication

# 4. Creating an Authentication Service

To manage all Firebase Authentication operations efficiently, we'll create a dedicated AuthService class. This service will handle user registration, login, logout, and other authentication-related tasks.

# AuthService.swift

import Foundation
import FirebaseAuth

@MainActor
class AuthService: ObservableObject {
  @Published var user: User?
  @Published var isSignedIn: Bool = false
  @Published var authError: String?
  
  static let shared = AuthService()
  
  private init() {
    self.user = Auth.auth().currentUser
    self.isSignedIn = self.user != nil
    _ = Auth.auth().addStateDidChangeListener { [weak self] _, user in
      self?.user = user
      self?.isSignedIn = user != nil
    }
  }
  
  // MARK: - Email & Password Authentication
  
  func register(email: String, password: String) async -> Bool {
    do {
      let result = try await Auth.auth().createUser(withEmail: email, password: password)
      self.user = result.user
      self.isSignedIn = true
      return true
    } catch {
      self.authError = error.localizedDescription
      return false
    }
  }
  
  func login(email: String, password: String) async -> Bool {
    do {
      let result = try await Auth.auth().signIn(withEmail: email, password: password)
      self.user = result.user
      self.isSignedIn = true
      return true
    } catch {
      self.authError = error.localizedDescription
      return false
    }
  }
  
  func logout() async -> Bool {
    do {
      try Auth.auth().signOut()
      self.user = nil
      self.isSignedIn = false
      return true
    } catch {
      self.authError = error.localizedDescription
      return false
    }
  }
}

# AuthView.swift

struct AuthView: View {
  @StateObject private var authService = AuthService.shared
  @State private var email = ""
  @State private var password = ""
  
  var body: some View {
    VStack {
      if authService.isSignedIn {
        Text("Welcome, \(authService.user?.email ?? "User")")
        Button("Logout") {
          Task {
            await authService.logout()
          }
        }
      } else {
        TextField("Email", text: $email)
          .textFieldStyle(RoundedBorderTextFieldStyle())
        SecureField("Password", text: $password)
          .textFieldStyle(RoundedBorderTextFieldStyle())
        Button("Login") {
          Task {
            let success = await authService.login(email: email, password: password)
            if !success {
              print(authService.authError ?? "Unknown error")
            }
          }
        }
        Button("Register") {
          Task {
            let success = await authService.register(email: email, password: password)
            if !success {
              print(authService.authError ?? "Unknown error")
            }
          }
        }
      }
    }
    .padding()
  }
}

# Explanation:

  • Singleton Pattern: AuthService is implemented as a singleton to ensure a single instance manages authentication across the app.
  • Published Properties: user, isSignedIn, and authError are published to allow SwiftUI views to reactively update based on authentication state.
  • Email & Password Methods: register, login, and logout methods handle respective authentication tasks.
