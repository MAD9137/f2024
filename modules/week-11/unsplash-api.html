# Overview

- From https://unsplash.com/about

Beautiful, free images gifted by the world’s most generous community of photographers. Better than any royalty free or stock photos.

As part of assignment 2, you can fetch a random image from https://source.unsplash.com/random and use the url as one of 20 that make up the list of photo urls. However, if you inspect the url of the returned image, there's a number of query parameters the Unsplash API adds that ultimately affects the photo displayed in your app.

For Example:

https://images.unsplash.com/photo-1666559821958-3ca7a8db8a39?crop=entropy&cs=tinysrgb&fit=max&fm=jpg&ixid=MnwxfDB8MXxyYW5kb218MHx8fHx8fHx8MTY2ODUzMzU4NQ&ixlib=rb-4.0.3&q=80&w=1080

if we decompose the url down into its components we can see how the photo has been processed/modified before responding to our request:

https://images.unsplash.com/

// The photo resource and identifier
photo-1666559821958-3ca7a8db8a39

// Crop mode controls how the image is aligned
// when fit=crop is set. The w and h parameters
// should also be set, so that the crop behavior
// is defined within specific image dimensions.
?crop=entropy

// The cs parameter specifies the color space of the output image.
&cs=tinysrgb

// controls how the output image is fit to its target 
// dimensions after resizing, and how any background
//areas will be filled.
&fit=max

// Output format
&fm=jpg

// required per API guidelines
&ixid=MnwxfDB8MXxyYW5kb218MHx8fHx8fHx8MTY2ODUzMzU4NQ

// Not quite sure but likely refers to version of backing processes
&ixlib=rb-4.0.3

// Controls the output quality of lossy file formats
// (jpg, pjpg, webp, or jxr).
&q=80

// width %
&w=1080
