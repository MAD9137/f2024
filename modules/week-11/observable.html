# Overview

Using the InternetConnected class from the previous page we can refactor it so it can be bound to SwiftUI views. While the UpdatePathHandler updates our connected property anytime a change in network state occurs it can't be used to trigger updates in our UI.

We'll need to update our class definition so that class instances can take advantage of automatic view updating as values change.

# Define an Observable Object with ObservableObject

To refactor our network monitor, we first need to update our class definition to adopt the ObservableObject protocol.

class InternetConnection: ObservableObject {
        // code in here...
}

# Publish Properties for Subscription

The @Published attribute indicates that the value associated with a property will be published for subscription.

class InternetConnection: ObservableObject {

    @Published var isConnected = false
    @Published var isExpensive = false

        //rest of code in here...
}

# Declare an Observable Instance with @StateObject

From the docs:

A property wrapper type that instantiates an observable object.

Now that we've refactored our InternetConnection class, we can create observable instances with the @StateObject property wrapper.

@StateObject var myConnection = InternetConnection()
1

Building on the above we can now observe changes to the isConnected property and trigger a prompt notifying users when connectivity has been interrupted.

   // Create an instance
    @StateObject var myConnection = InternetConnection()
    
    // Declare Alert presentation state variable
    @State var presentAlert: Bool = false
    
    var body: some View {
        VStack {
            // Views in here
            
            // when the connection state changes
        }.onChange(of: myConnection.isConnected, perform: { connectionStatus in
            
            // if connection state is false present alert to user
            if connectionStatus == false {
                presentAlert = true
            }
        })
        // Alert configuration
        .alert("Alert Title", isPresented: $presentAlert, actions: {
            Button("OK", role: .none , action: {
                // once the user interacts with the alert
                // it should be dismissed even without the
                // next line.
                presentAlert = false
            })
        }, message: {
            Text("Connection to the network has been lost")
        })
    }

# See Also

There's some nuance around when to use @StateObject and @ObservedObject. This nuance is mainly around the idea of which view owns the data being observed. Generally, whichever view is first to create an object should use @StateObject while subsequent views use @ObservedObject.
