# Assignment A2

Weather App in SwiftUI

# Objective

Build a feature-rich weather app in SwiftUI to enhance your skills in building multi-screen applications, working with APIs, managing state, and creating dynamic user interfaces.

# Requirements

  1. Main Features:

    • City List Screen:

      • Displays a list of cities.
      • Users can add selected cities to the list.
      • Each item in the list must include:
        • City name.
        • Current temperature. (should update every time interval in settings)
        • Current weather
        • Weather icon.
        • Local time. (should update every minute)

    • Search Screen:

      • Allows users to search for cities.

    • Settings Screen:

      • Contains at least two options:
        1. Time Refresh Interval: Allows users to set how often the app refreshes weather data.
        2. About: Navigates to the About screen.

    • About Screen:

      • Displays information about the app and the author.

    • City Weather Detail View:

      • Displays detailed current weather conditions
        • Temperature
        • UV index
        • Wind Speed
        • Humidity
      • and a forecast for the next few hours.

  1. Data Handling:

    • Use a weather API of your choice (e.g., OpenWeatherMap) to fetch weather data.
    • Ensure proper error handling for API calls.

  2. UI and UX:

    • Use SwiftUI to create an intuitive and visually appealing design.
    • Ensure your app has an icon for dark and tinted mode.App Icons (opens new window)
    • Make sure your app supports dark mode eficiently.Supporting Dark Mode in Your Interface (opens new window)
    • The app should be navigable using a tab bar or navigation view.
    • You can choose the app accent color you want as well as details: icons, font, etc.
    • The city detail view needs to have the map in the background
    • The city needs to be discerneable. Find a good zoom level
    • The screen should have a translucid efect over it
    • The color this overlay is dependable of the temperature. Cold tends to blue and hot to red. (Check this repo Repo (opens new window))
    • In the detail view, taping on the temperature switch from Farenheit to Celsius and vice-versa.
    • In the about view, add a hidden feature where, if you tap your picture 3 times it will show your picture as a kid or something like that.

  3. Code Requirements:

    • Write clean, modular, and well-documented code.
    • Use @State, @Binding, and @EnvironmentObject where applicable.
    • DO NOT USE CLOSURES for async methods. Use async/await instead.
    • Follow best practices for SwiftUI app development.

  4. Bonus Points:

    • Use animations to enhance the UI.
    • Allow users to rearrange the city list.

# Deliverables

  • Push the project to your GitHub account and share the Github URL (not the origin URL, please)).
  • Include a README.md with:
    • A brief description of the app.
    • API key management details (if required).
    • Any additional features or assumptions.

# Grading Criteria

  • Functionality (40%): Does the app meet the requirements?
  • Code Quality (30%): Is the code clean, readable, and modular?
  • UI/UX Design (20%): Is the app intuitive and visually appealing?
  • Creativity and Extras (10%): Any additional features or enhancements.
